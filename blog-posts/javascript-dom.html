JavaScript and the DOM Explained

Here is my breakdown of the core JavaScript concepts from this sprint. I am keeping it simple and explaining things the way they make sense to me.

1. HTML, CSS, and JavaScript

Think of building a website like building a house.

HTML is the raw structure. It is the walls, floors, and doors. Without it you just have an empty lot.

CSS is the interior design. It is the paint color, the carpet, and the furniture layout. It makes the bare structure look good.

JavaScript is the electricity and plumbing. A house looks nice with just HTML and CSS, but JavaScript makes it functional. It is what happens when you flick a light switch or turn on a tap.

2. Control Flow and Loops

Control flow is how a computer makes decisions. Think about doing laundry. If the clothes are dirty, you wash them. If they are clean, you fold them. That is control flow.

A loop is just repeating a task until a condition is met. When you are washing a pile of dishes, you do not make a new plan for every single plate. You just run a loop: while there are dishes in the sink, wash the next one. 

while (sink.hasDishes == true) {
  // Keep doing this until the sink is empty
  washDish();
}

3. The DOM

The DOM stands for Document Object Model. It is basically a map of your website that the browser creates when the page loads.

JavaScript uses this map to find specific HTML elements and change them on the fly. If you want to change the text on a button when someone clicks it, you do not reload the entire webpage. JavaScript uses the DOM to grab just that one button and update the text instantly. 

// Finding a title on the page and changing its text
document.getElementById('mainTitle').innerHTML = 'New Title';

4. Arrays and Objects

These are just two different ways to store data in code. The difference is how you find the data later.

An Array is a basic list where the order matters. Think of a queue of people at a store. If you want to find someone, you look for their position in the line.

An Object is a collection of details where the order does not matter. Think of a contact list in your phone. You do not look for your friend by searching for the 15th person in your phone. You just search directly for their name to get their number. 

// ARRAY: Accessing by position
let line = ['John', 'Sarah', 'Dave'];
console.log(line[0]); // This gets John

// OBJECT: Accessing by name
let phonebook = { John: 123456, Sarah: 987654 };
console.log(phonebook.John); // This gets 123456

5. Functions

A function is a saved set of instructions. If you have a specific task you need to do all the time, you do not want to write the code out line by line every single time.

Instead you pack all those steps into a block, give it a name, and just call that name. It is like a recipe. Once you write down how to bake a cake, you just follow the recipe instead of inventing it from scratch every time you want dessert.

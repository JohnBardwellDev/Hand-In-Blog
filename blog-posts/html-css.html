CSS Positioning: Relative, Absolute and Fixed

April 3, 2026

When you build a webpage, every element just stacks on top of each other from top to bottom by default. CSS positioning lets you break out of that and place things exactly where you want them. There are three main values you need to know: relative, absolute, and fixed.

Relative

Relative positioning moves an element relative to where it would normally sit. Think of it like nudging something out of its usual spot but the space it originally took up is still reserved. 

.box {
  position: relative;
  top: 20px;
  left: 10px;
}

This moves the box 20px down and 10px to the right from where it would normally be. Everything else on the page stays put.

Absolute

Absolute positioning removes the element completely from the normal flow of the page. It gets placed relative to its nearest parent that has a position set on it. If no parent has a position, it uses the whole page. 

.parent {
  position: relative;
}

.child {
  position: absolute;
  top: 0;
  right: 0;
}

This puts the child element in the top right corner of its parent. Unlike relative, the space the child used to occupy is gone and other elements fill in around it.

Fixed

Fixed positioning locks an element to the browser window itself. It stays in the same spot even when you scroll. This is how sticky navigation bars and floating buttons work. 

.navbar {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
}

No matter how far down the page you scroll, this element stays pinned to the top of the screen.

When to use each one

Use relative when you want to nudge something slightly without affecting the layout around it. Use absolute when you need to place something precisely inside a container, like a badge or a tooltip. Use fixed when you want something to stay visible no matter where the user scrolls, like a header or a back to top button.
